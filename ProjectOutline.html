Federated Learning POC - Project Structure

aggregator/
Contains the server responsible for managing federated learning, receiving model weights from hospitals, and aggregating them to form the global model.
server.py
Handles the uploading of model weights and aggregation process to generate the global model using federated learning.
data/
Contains the CSV files with the hospital patient data used for local training.
hospital_1_patients.csv
Data for Hospital 1, including patient details like age, glucose levels, and medical history.
hospital_2_patients.csv
Data for Hospital 2, formatted similarly to Hospital 1.
hospital_3_patients.csv
Data for Hospital 3, structured like the other hospitals.
demo/
Contains the demo file to simulate the federated learning process and test the predictions.
demo_federated_learning.py
Runs the demo for federated learning, simulating model training and predictions using the aggregated model.
hospitals/
Contains scripts for training models on local hospital data and extracting model weights.
hospital_1.py
Script for training a model on Hospital 1's data and extracting the model weights.
hospital_2.py
Script for training a model on Hospital 2's data and extracting the model weights.
hospital_3.py
Script for training a model on Hospital 3's data and extracting the model weights.
models/
Contains the model weights for each hospital as well as the global model after aggregation.
global_model.pth
The global model that represents the aggregated knowledge from all hospitals.
hospital_1_weights.pth
Model weights for Hospital 1.
hospital_2_weights.pth
Model weights for Hospital 2.
hospital_3_weights.pth
Model weights for Hospital 3.
patient_demo_app.py
A demo app to show how the aggregated model can be used for making predictions based on user input.
templates/
Contains the HTML template used to display the demo app.
index.html
HTML template for the user interface of the patient demo app.
